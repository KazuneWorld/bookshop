# 書籍ショッピングECサイト
* **制作時期:** 2026年1月（期間: 約1か月間）
* **開発体制:** チーム開発（5名）

学校のグループ演習を起点に開発した、JSP/Servletによる動的なWebベースのECサイトです。
提示された基本要件を満たすだけでなく、自宅開発にて多くの独自機能（ランキング、在庫連動など）の追加、MVCモデルへのリファクタリング、さらにはAWSへのデプロイまで自発的に行いました。

## デプロイ環境（AWS）
クラス発表の際、実際にインターネット経由で動作を実演するため、インフラ環境の構築を独自に行いました。
* **インフラ:** AWS（EC2, RDS）を使用し、動的なWebアプリケーション環境を構築。
* ※現在は安全のためインスタンスを停止しています

## ✨ こだわった部分 ✨

### 1. 要件定義を超えた実用的な独自機能の追加
ユーザーの購買意欲を高め、実際のECサイトに近づけるために以下のオリジナル機能を実装しました。
* **人気ランキング表示機能:** 注文履歴のテーブルから、SQLの `SELECT` 文を用いて売上数の多い書籍を抽出し、トップページにランキング形式で動的表示しました。
* **カテゴリ検索機能:** 書籍データベースにカテゴリを追加し、目的の本を探しやすくなるよう絞り込み機能を追加しました。
* **残在庫数表示:** 在庫が少なくなった際、商品詳細ページにリアルタイムな残在庫数を表示し、ユーザーへ購買を促す仕組みを実装しました。

### 2. MVCモデルを意識したリファクタリング
チーム開発の過程でコードの肥大化と乱雑化（サーブレットの乱立、SQLの散在）が発生したため、コードの可読性と保守性を向上させるために重点的にリファクタリングを行いました。
* **サーブレットの集約:** 乱立していたサーブレットを適切な粒度に統合・整理しました。
* **DAOパターンの徹底:** 各地に散らばっていたSQL文を、データ操作を専門に行う正しいDAO（Data Access Object）クラスへと移譲しました。
* **MVCモデルの適用:** ビジネスロジックをModel（Javaクラス）へ分離し、View（JSP）、Controller（Servlet）の役割分担を明確にしました。

### 3. 反省点から得た時間管理・役割分担の学び
今回の開発では、追加機能の充実を優先するあまり、後半の時間配分やチーム内での正確な役割分担・連携に苦労するという反省点がありました。
* **対策と工夫:** デザイン（CSS）の原型作成にはAIツールを効率的に活用し、そこから自分たちでブラッシュアップをすることで開発スピードをカバーしました。この経験から、開発初期段階における計画の優先順位付けの重要性を深く学びました。

### 4. チームの強みを活かしたUI/UX（オリジナルデザイン）
* **フロントエンドのクオリティ向上:** チームメンバーが完全オリジナルのロゴやアイコンのデザインを作成してくれたため、それをシステムに組み込みました。バックエンドの機能だけでなく、見た目のデザインも含めて「本物のWebサイト」のような高いクオリティに仕上げることができました。

## 技術スタック

### バックエンド / フロントエンド
* **言語:** Java 21, HTML5, CSS3, JavaScript
* **Web技術:** JSP & Servlet（HttpSessionを用いたログイン状態・ショッピングカートの管理）
* **Webサーバー:** Apache Tomcat 10

### データベース / インフラ
* **学校環境:** MySQL (XAMPP環境)
* **自宅・本番環境:** Amazon RDS (MySQL), Amazon EC2
* **開発環境（IDE）:** Eclipse, Visual Studio Code

## 担当機能
* トップページ（人気ランキング、カテゴリ検索、ポップアップ広告など）のバックエンド処理・DB連携
* 商品詳細ページ（残在庫連動、書籍画像表示）の作成
* 商品追加・編集ページの作成
* プロジェクト全体のコードレビューおよびリファクタリング（MVCモデルを意識した設計、DAO化）
* **インフラ構築・デプロイ:** AWS（EC2, RDS）を用いたWebアプリケーション環境の設計・構築（完全単独担当）

## 環境構築・実行方法

### 1. 前提条件
* **JDK:** 21
* **Webサーバー:** Apache Tomcat 10
* **データベース:** MySQL

### 2. データベースの準備
1. MySQLに接続し、データベース`bookshop`を作成します。
2. リポジトリの直下にあるテーブル構築用のSQLファイル`bookshop.sql`を実行し、必要なテーブル構造および初期サンプルデータをインポートします。

### 3. ソースコードのインポートと接続設定
1. お使いのIDEにプロジェクトをインポートします。
2. プロジェクトのビルドパスに **Tomcat 10** のランタイムが設定されていることを確認します。
3. データベースへの接続情報（接続URL、ユーザー名、パスワード）が記述されている`DAO` パッケージ内のJavaファイルを開き、ご自身の環境の設定に合わせて書き換えてください。

### 4. 実行方法（ローカルサーバーでの起動）
1. IDE上でプロジェクトを `サーバーで実行` します（またはTomcatサーバーにプロジェクトを追加します）。
2. Tomcatサーバーを起動します。
3. ブラウザを開き、以下のURLにアクセスすることでECサイトのトップページが表示されます。
   ```text
   http://localhost:8080/bookshop/Main
   ```
